Q.1.0:How to compile gfitn?

  1. Create your own source directory and copy all files in /usr/local/gfitn/examples to this directory.
  2. Modify functions.c or create another source file and include them in you makefile.
  3. Mke sure that you new added functions are included in tabfunc.c functions list.
  4. Run make gfitn install-bin DESTDIR=/home/home_dir.
  5. If you get an error when linking gfitn like "/usr/lib/libf2c.so: undefined reference to `MAIN__'", include the following lines in one of your source
    MAIN__()
    {}
Q.1.1: When do I need to fit more than one data set at the same time?

When your model fit equation depends on more than just one independent variable. The user can use the --xyt option to fit more than one data files with the t value as the second independent variable. It is possible to perform model fits in n-folder spaces of independent variables but the user functions have to written in a more complicated way. Ask Pedro Sebastião how to do it?

Q.2.1: I was able to used Gnuplot to draw the plots but not Xmgr. Sugestions?

See that you have compiled your sources with libgfitn.a version >= 4.1.0.

Q.2.2: I set the xscale, yscale to log but get linear scales when using xmgr. Sugestions?

Check your data or gph files for x=0 oy y=0. Gnuplot dosen't care but Xmgr does.

Q.3.1: How do I start xfitenv with a particular file.env?

Just start xfitenv with file.env as an argument. Example:
prompt# xfitenv file.env

Q.3.2: I get no Output when I try to fit!

  • See if you have the correct bin file.
  • Check the output generated by xfitenv and GFITN.
  • Check if you have set the correct GFITN option to read your data file.

Q.3.3: I try to plot and get noting!

  • See if you have the correct bin file.
  • Check if you have the correct GFITN options in Setup file.
  • Look at the GFITN output.
  • Try to run GFITN by hand without the xfitenv interface to check for additional output.
