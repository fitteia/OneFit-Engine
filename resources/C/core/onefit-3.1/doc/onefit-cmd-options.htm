The -@ option allows the user to pass options to gfitn through an options file like this:
#--device=hpdj
#--Outdev=| lpr -P lince
#--device=postscript
#--Outdev=| lpr
#--device=postscript
#--Outdev=| lpr -P lps3
--xyt
--autox
--autoy
--nolabel
--xscale=log lin
--yscale=logT1 lin
--ptype=point errbar
#--num=200
# freq graphs
--xm=10 0
--xM=4e8 180
--ym=0.001 0
--yM=1000 2
#opcoes malucas
#--ym=0.001
#--yM=300
#--ptype=errbar
# ang graphs
#--xm=0
#--xM=180
#--ym=0
#--yM=2
#--xscale=lin
#--yscale=lin
#--ptype=errbar
--gph=my_file.gph

The # is used as a comment character and the rest of the line is omitted.

One should note the following:

  • --xy
    takes no arguments
    expects that the user data file has the following format:
    ###############################################
# comment line
#
# more comment lines

x1 y1 ey1
x2 y2 ey2
.  .  .
.  .  .
# comment line
##############################################
  • --xyt
    takes no arguments
    expects that the user data file has the following format:
    ###############################################
# comment line
#
# more comment lines
   t
x1 y1 ey1
x2 y2 ey2
.  .  .
.  .  .
# comment line
##############################################

    Please note that the first non-comment line will have to contain one number which will be passed to user functions as the value t.

  • The xscale,yscale can have the following values:
    • lin
    • log
    • logT1
    • inv
    --yscale=logT1 means the log(1/y)
    --yscale=inv means 1/y
  • --ptype can be:
    • point
    • errbar
    errbar means that points are plotted with error bars. When using xmgr gfitn forces the use of errbar so is pointless to define it as point.
  • --gph allows the user to choose the names of the files containing the theoretical curves

In all cases each option can take more than one argument but never less than the number of data files being fitted at the same time (see FAQ).
