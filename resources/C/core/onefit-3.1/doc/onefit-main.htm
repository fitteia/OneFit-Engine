Foreword

OneFit results of several years of experience in model fitting to Nuclear Magnetic Relaxation experimental results. In spite of being a particular field, the fact that in most cases the exact model is not known and that different theoretical models have to be tried in the analysis of the experimental results, made me realise that the accumulated experience could be useful to others. OneFit is an evolution of former GFITN package.

OneFit brings, perhaps, noting new to field of numerical analysis, function minimization, and model fitting, but, I think, it makes life easier for those who have to deal with not-so-simple non-linear functions in model fitting. OneFit integrates the compilling, numerical, and graphical facilities provided by specialized software packages, each one optimized for a particular task. OneFit uses the gcc compiler (OneFit kernel), CERNLIB (OneFit minimization routines), perl/perl-Tk (OneFit GUI), Xmgr/Grace (graphics), ImageMagick (convert/display different image formats), your favorite spread-sheet editor to process data files, to mention the main packages.

I hope you find it useful. Remember, however, that I decline all responsability in whataver happens in result of the use of OneFit.

General Introduction

Model fitting is almost like an artistic activity since is demands much more than some numerical routines to perform the task of find some general set of parameters which brings the theoretical models in a close match to the experimental results.
In fact in order to interpret some experimental results one needs a theoretical model which we can write as a linear sum of different partial equations: y = a1*f1(p1,..pm,x1,..xn) + a2*f2(p1,..,pm,x1,..,xn) + ... +al*fl(p1,..,pm,x1,..,xn) where we can identify a1,a2,a3,...al as linear parameters of the model (not very difficult to estimated), p1,..,pm non-linear parameters to be estimated (a more difficult task) and x1,..,xn a point in n-folder space of independent variables. In order to solve the problem of finding the a1,..al,p1,..,pm values we need to have at least m+l+1 points in the n-folder. space.

A normal user will need to have an easy way of change the scale of his problem, that is to change the value of m+l of the parameter's space an also the n-folder space where he's working. OneFit helps the user on this task.

Before going into the Tutorial let me just tell you a few words on the OneFit/GFITN kernel. I call it kernel because the user has to link his C-source file with the necessary implementation of functions f1,..fl with some libraries containing the envelope which makes everything easier. This envelope deals with the Chi-square minimization, graphics, input/output, etc. The Chi-square minimization is done by calling a CERN Library routine called minuit.

Tutorial

Let's suppose the user has some experimental results which he wants to interpret in terms of some theoretical function f(x)=a1+a2*exp(-b*x). We first note that he's working in a one dimension space of independent variables x, and in a 3-folder parameter space. Therefore, he needs at least 4 values of x. Now the user has to decide how to analyse the expression of y. He can think on y as an expression with 3 non-linear parameters a1,a2,b or as the linear sum of functions f1=1 and f2=exp(-b*x) where a1,a2 are two linear parameters and b is a non-linear parameter. There two ways of looking at expression f(x) represent two different ways of programming the model fit to the experimental results.

User Interfaces to OneFit

OneFit kernel processes several files (datafiles, parameter files (in case of non-linear functions), output files, setup files log files. The user can do this by hand, for example if one wants to call onefit from within another programme (It may happens, you know!). Another, easier way is to interact with OneFit using the xonefit GUI interface. xonefit is a perl/Tk script which, I hope is self-evident on ist use. The files associated with one particular fitting session together with the OneFit kernel and the user fitting model are called the Fitting Environment.
